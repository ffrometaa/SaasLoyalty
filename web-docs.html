Registro Multi-Paso (/register)
1
Datos del Negocio Nombre del negocio (required), tipo de negocio (dropdown, required), slug para la URL del tenant (required, único)
2
Selección de Plan Toggle mensual/anual (17% descuento en anual), 3 planes disponibles: Starter $79, Pro $199, Scale $399. Order summary lateral con pricing calculado.
3
Credenciales + Pago Email (required, validado), contraseña, aceptación de términos. Botón → Stripe Checkout. Trial 14 días mencionado. POST /api/register crea checkout session.
✓ El flujo de Enterprise va por /contact — no tiene registro self-service. Lleva a formulario de demo/contacto con equipo de ventas.

API Routes
POST
/api/register
Recibe: businessName, businessType, slug, email, userId, plan, billingPeriod. Crea sesión de Stripe Checkout. Retorna checkout URL para redirigir al pago.
GET
/api/auth/cross-domain-login
Establece sesión en dashboard.loyalbase.dev después del login en la web. Necesario porque son dominios distintos (no comparten cookies).
POST
/api/webhooks/stripe
Handler de eventos de Stripe: checkout completado, suscripción creada, pago fallido, cancelación. Actualiza plan_status en tabla tenants.
GET
/api/invite/[token]
Valida el token de invitación al equipo. Retorna info del invite (email, tenant, role) para pre-rellenar el formulario.
POST
/api/invite/accept
Procesa la aceptación de una invitación de equipo. Crea el registro en tenant_users y marca el invite como aceptado.
